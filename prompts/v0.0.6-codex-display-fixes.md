## F7 🟠 ILI9341/ST7789 MADCTL 适配 Kconfig rotation

**问题**: `CONFIG_ROBOMIND_DISPLAY_ROTATION`(Kconfig) 仅 GC9A01 分支引用，ILI9341/ST7789 硬编码 MADCTL。

```cpp
// ❌ ILI9341 当前 (硬编码):
LcdWriteCommand(0x36);
LcdWriteData(0x48);  // MX|BGR, 固定值忽略 rotation

// ✅ 修复:
LcdWriteCommand(0x36);
#if CONFIG_ROBOMIND_DISPLAY_ROTATION == 0
    LcdWriteData(0x48);  // Portrait: MX|BGR
#elif CONFIG_ROBOMIND_DISPLAY_ROTATION == 1
    LcdWriteData(0x88);  // Landscape: MV|MX|BGR
#elif CONFIG_ROBOMIND_DISPLAY_ROTATION == 2
    LcdWriteData(0x28);  // Portrait inverted: MY|BGR
#else
    LcdWriteData(0xE8);  // Landscape inverted: MV|MX|MY|BGR
#endif
```

ST7789 同理，将其 `0x00` 替换为 rotation 适配值 (ILI9341 和 ST7789 MADCTL 编码相同)。

`Initialize()` 中约第 128-135 行:


## F24 🟡 #ifndef CONFIG_* 后备宏去重

`display_driver.cpp` 第 42-69 行有 13 个 `#ifndef CONFIG_*` 后备宏。
Kconfig `default` 字段已提供相同默认值 → 双重定义，可能漂移不一致。

**方案**: 保留 `#ifndef` 作为编译安全网，但在每个后面加注释标注 Kconfig 默认值:

```cpp
#ifndef CONFIG_ROBOMIND_DISPLAY_WIDTH
#define CONFIG_ROBOMIND_DISPLAY_WIDTH 240  // Kconfig default: 240
#endif
```

无需删除——这是防御性编程，有存在价值。但确保每个 `#ifndef` 后备值与 Kconfig 的 `default` 一致。

**检查项**: 逐项对比 13 个 `#ifndef` 值与 `Kconfig.projbuild` 中对应的 `default` 值。如有不一致报告。
