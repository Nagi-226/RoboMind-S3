### C3 🔴 chat_engine.cpp — 将 HTTP 请求移入独立 FreeRTOS Task

文件: `main/chat_engine.cpp`

**当前问题**: `DoChatRequest()` 直接在调用线程中阻塞执行 `esp_http_client_perform`，
导致 LVGL UI 冻结。`.h` 注释说"在独立 task 中执行"但代码未实现。

**修复方案**: 创建独立 task 运行 HTTP 请求，SendMessage 通过信号量通知。

实现步骤:

1. 在 .cpp 顶部添加 task 相关静态变量:
```cpp
static TaskHandle_t s_chat_task = nullptr;
static std::string s_pending_message;  // 待发送消息
static SemaphoreHandle_t s_request_sem = nullptr;  // 请求信号量
```

2. 新增静态 task 函数:
```cpp
static void ChatNetworkTask(void* arg) {
    auto* engine = static_cast<ChatEngine*>(arg);
    while (true) {
        // 等待 SendMessage 通知
        if (xSemaphoreTake(s_request_sem, portMAX_DELAY) == pdTRUE) {
            std::string msg = s_pending_message;
            s_pending_message.clear();
            
            // 将用户消息加入历史 (engine 内部处理)
            engine->DoChatRequest();
            
            // 请求完成
            engine->SetBusy(false);
        }
    }
}
```

3. 在 `ChatEngine::Initialize()` 中创建 task 和信号量:
```cpp
s_request_sem = xSemaphoreCreateBinary();
xTaskCreatePinnedToCore(ChatNetworkTask, "chat_net", 8192,
                        this, 4, &s_chat_task, 1);  // Core 1
```

4. 修改 `SendMessage()` — 不再直接调用 DoChatRequest:
```cpp
bool ChatEngine::SendMessage(const std::string& user_text) {
    if (busy_) {
        ESP_LOGW(TAG, "ChatEngine busy");
        return false;
    }
    
    last_user_message_ = user_text;
    history_.push_back({"user", user_text});
    TrimHistory();
    
    busy_ = true;
    if (status_callback_) {
        status_callback_(Status::kThinking, "Thinking...");
    }
    
    // 通知网络 task 执行请求
    s_pending_message = user_text;
    xSemaphoreGive(s_request_sem);
    
    return true;
}
```

5. 需要添加公开方法让 task 设置 busy 状态，或者在 .cpp 中用 friend/static 访问:
   - 方案 A: 添加 `void SetBusy(bool b) { busy_ = b; }` 公开方法
   - 方案 B: 在 ChatNetworkTask 中直接通过 `engine->busy_ = false` 访问（需要 friend）

   推荐方案 A。

6. 在 `DoChatRequest()` 末尾，移除设置 `busy_ = false` 的代码 (改由 ChatNetworkTask 设置)。

### H2 🟠 chat_ui.cpp — SSE 流式改为更新同一个气泡

文件: `main/chat_ui.cpp:214-224, 226-263`

**当前问题**: 每条 SSE delta 创建一个新气泡。50 条 delta = 50 个气泡叠加。

**修复方案**: 首次创建时保存指针 → 后续只更新文本和高度。

```cpp
// 在 chat_ui.h 的 private 区添加成员:
lv_obj_t* pending_bubble_{nullptr};  // 流式更新中的气泡容器
lv_obj_t* pending_content_label_{nullptr};  // 对应的内容 label

// 修改 AppendMessage:
void ChatUI::AppendMessage(const std::string& role, const std::string& content) {
    if (role == "assistant") {
        pending_assistant_text_ += content;
        if (pending_bubble_) {
            // 更新现有气泡的文本和高度
            lv_label_set_text(pending_content_label_, pending_assistant_text_.c_str());
            lv_obj_update_layout(pending_content_label_);  // 强制布局计算
            lv_obj_set_height(pending_bubble_, lv_obj_get_height(pending_content_label_) + 24);
            lv_obj_scroll_to_view(pending_bubble_, LV_ANIM_OFF);
        } else {
            // 首次: 创建气泡并保存指针
            pending_bubble_ = AddMessageBubble("assistant", pending_assistant_text_);
            // AddMessageBubble 需要改为返回 lv_obj_t*
        }
    } else {
        AddMessageBubble(role, content);
    }
}
```

需要同步修改 `AddMessageBubble` 返回类型为 `lv_obj_t*`，并保存 `content_label` 指针。

在 SSE 流结束时（非 assistant 消息到达时）重置:
```cpp
pending_bubble_ = nullptr;
pending_content_label_ = nullptr;
pending_assistant_text_.clear();
```

> ⚠ 如果这个改动太复杂，先做简化版：在 AddMessageBubble 中删除前一个 assistant 气泡再创建新的。至少避免气泡无限叠加。
