## F1 🔴 display_driver.cpp — LVGL flush CASET/RASET 的 DC 引脚错误

**根因**: v0.0.3 修复把 CASET 命令+4 个坐标字节合并为单次 5 字节 SPI 事务。
但 DC 引脚在整次事务中保持 DC=0（命令模式），导致后 4 个坐标字节被 ILI9341 当作命令而非数据。

**ILI9341 数据手册**: DCX='0' = 命令, DCX='1' = 数据。每个字节都受 DCX 限定。

**错误代码**（v0.0.3 修复后的状态）:
```cpp
// ❌ 整个 5 字节事务中 DC=0
gpio_set_level(static_cast<gpio_num_t>(CONFIG_ROBOMIND_DISPLAY_PIN_DC), 0);
spi_transaction_t t_caset = {};
t_caset.length = sizeof(caset_cmd) * 8;  // 40 bits = 5 bytes
t_caset.tx_buffer = caset_cmd;            // {0x2A, x1_hi, x1_lo, x2_hi, x2_lo}
spi_device_transmit(s_spi_device, &t_caset);
// 后 4 个坐标字节全部被当作命令！
```

**正确修复**——参考同文件中 `LcdWriteCommand` / `LcdWriteData16` 的模式:

```cpp
// ✅ 事务 1: DC=0, 发送命令 0x2A
gpio_set_level(static_cast<gpio_num_t>(CONFIG_ROBOMIND_DISPLAY_PIN_DC), 0);
uint8_t caset_cmd = 0x2A;
spi_transaction_t t1 = {};
t1.length = 8;
t1.tx_buffer = &caset_cmd;
spi_device_transmit(s_spi_device, &t1);

// ✅ 事务 2: DC=1, 发送 4 字节坐标数据
gpio_set_level(static_cast<gpio_num_t>(CONFIG_ROBOMIND_DISPLAY_PIN_DC), 1);
uint8_t caset_data[4] = {
    static_cast<uint8_t>((area->x1 >> 8) & 0xFF),
    static_cast<uint8_t>(area->x1 & 0xFF),
    static_cast<uint8_t>((area->x2 >> 8) & 0xFF),
    static_cast<uint8_t>(area->x2 & 0xFF),
};
spi_transaction_t t2 = {};
t2.length = 32;
t2.tx_buffer = caset_data;
spi_device_transmit(s_spi_device, &t2);
```

对 RASET (0x2B 命令) 做**完全相同**的拆分:
```cpp
// RASET 命令 0x2B
gpio_set_level(..., DC, 0);
uint8_t raset_cmd = 0x2B;
// ... 事务发送命令

// RASET 数据 (y1_hi, y1_lo, y2_hi, y2_lo)
gpio_set_level(..., DC, 1);
uint8_t raset_data[4] = {...};
// ... 事务发送数据
```

**然后 RAMWR 命令保持不变** (DC=0 发 0x2C，DC=1 发像素数据)。

**删除**: caset_cmd[5] 数组、raset_cmd[5] 数组（不再需要合并结构）。

## F2 🔴 chat_engine.cpp + Kconfig — temperature string→number

**根因**: Kconfig `ROBOMIND_AI_TEMPERATURE` 类型为 `string`，代码用 `cJSON_AddStringToObject`。
OpenAI API 规范要求 JSON number: `"temperature": 0.7` 而非 `"temperature": "0.7"`。

### F2a: 修改 Kconfig

文件: `main/Kconfig.projbuild`，约第 62-66 行。

将 temperature 从 string 改为 int（乘以 100 存整数，如 0.7 → 70）:

```kconfig
config ROBOMIND_AI_TEMPERATURE
    int "Temperature (×100)"
    default 70
    range 0 200
    help
        AI response temperature in hundredths (70 = 0.7).
        0 = deterministic, 200 = max creativity.
        Stored as integer to avoid floating-point Kconfig limitations.
```

### F2b: 修改代码

文件: `main/chat_engine.cpp`，约第 132 行。

```cpp
// 替换前:
cJSON_AddStringToObject(root, "temperature", CONFIG_ROBOMIND_AI_TEMPERATURE);

// 替换后:
float temperature = static_cast<float>(CONFIG_ROBOMIND_AI_TEMPERATURE) / 100.0f;
cJSON_AddNumberToObject(root, "temperature", temperature);
```

## F3 🔴 display_driver.cpp — InitLcdController 丢弃所有 SPI 返回值

**根因**: `InitLcdController()` 返回 `bool`，但内部 5+ 次 `spi_device_transmit()` 的 `esp_err_t` 返回值全部丢弃。函数**始终 return true**。SPI 总线故障时系统继续执行→花屏/崩溃。

**位置**: `InitLcdController()` 函数 (~233-356 行)。

**修复**: 在 `LcdWriteCommand`、`LcdWriteData`、`LcdWriteData16` 三个静态辅助函数中检查返回值:

```cpp
// 将三个辅助函数从 void 改为 bool:
static bool LcdWriteCommand(uint8_t cmd) {
    gpio_set_level(static_cast<gpio_num_t>(CONFIG_ROBOMIND_DISPLAY_PIN_DC), 0);
    spi_transaction_t t = {};
    t.length = 8;
    t.tx_buffer = &cmd;
    esp_err_t ret = spi_device_transmit(s_spi_device, &t);
    if (ret != ESP_OK) {
        ESP_LOGE(TAG, "SPI write command 0x%02X failed: %s", cmd, esp_err_to_name(ret));
        return false;
    }
    return true;
}

static bool LcdWriteData(uint8_t data) {
    gpio_set_level(static_cast<gpio_num_t>(CONFIG_ROBOMIND_DISPLAY_PIN_DC), 1);
    spi_transaction_t t = {};
    t.length = 8;
    t.tx_buffer = &data;
    esp_err_t ret = spi_device_transmit(s_spi_device, &t);
    if (ret != ESP_OK) {
        ESP_LOGE(TAG, "SPI write data 0x%02X failed: %s", data, esp_err_to_name(ret));
        return false;
    }
    return true;
}

static bool LcdWriteData16(uint16_t data) {
    gpio_set_level(static_cast<gpio_num_t>(CONFIG_ROBOMIND_DISPLAY_PIN_DC), 1);
    spi_transaction_t t = {};
    t.length = 16;
    t.tx_buffer = &data;
    esp_err_t ret = spi_device_transmit(s_spi_device, &t);
    if (ret != ESP_OK) {
        ESP_LOGE(TAG, "SPI write data16 0x%04X failed: %s", data, esp_err_to_name(ret));
        return false;
    }
    return true;
}
```

然后在 `InitLcdController()` 中逐处检查这些函数的返回值:

```cpp
// 在 InitLcdController() 的每个驱动分支中, 将:
LcdWriteCommand(0x01);
// 改为:
if (!LcdWriteCommand(0x01)) return false;

// 对所有 LcdWriteCommand / LcdWriteData / LcdWriteData16 调用都加上 if (!...) return false;
```

> ILI9341/ST7789 分支约 8 处调用需要加检查。
> GC9A01 分支约 50+ 处调用 —— 批量替换即可（用编辑器的查找替换功能）。

## F6 🔴 main.cpp — RunLvglLoop 无 watchdog 喂狗

**位置**: `main/display_driver.cpp`，`RunLvglLoop()` 函数 (~424-433 行)。

ESP-IDF 的 Task Watchdog Timer (TWDT) 默认 5 秒超时。如果 SPI 通信阻塞，
`lv_timer_handler()` 可能超过 5 秒不返回 → ESP32 硬复位。

**修复**: 两种方案，选方案 A（更轻量）。

**方案 A** — 循环内喂狗:
```cpp
void DisplayDriver::RunLvglLoop() {
    ESP_LOGI(TAG, "Starting LVGL loop");

    // 注册当前 task 到 interrupt watchdog
    esp_task_wdt_add(nullptr);

    const TickType_t period = pdMS_TO_TICKS(kTickIntervalMs);
    while (true) {
        esp_task_wdt_reset();  // ← 每轮喂狗
        vTaskDelay(period);
        lv_timer_handler();
    }
}
```

**方案 B** — 配置更长超时:
```cpp
// 在 Initialize() 中:
esp_task_wdt_init(30, false);  // 30 秒超时
```

选方案 A，更安全。

需要在文件顶部新增 include:
```cpp
#include "esp_task_wdt.h"
```
