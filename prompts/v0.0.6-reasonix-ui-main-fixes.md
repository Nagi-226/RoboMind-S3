## F10 🟠 ChatUI::Initialize() void → bool

**你审计发现的**: `Initialize()` 返回 void，内部 `lv_obj_create()` ×20+ 返回值全部未检查。
PSRAM 耗尽时 LVGL 静默失败→黑屏无提示。

**修复** (2 文件):

### chat_ui.h
```cpp
// 修改声明:
bool Initialize();  // 原: void Initialize();
```

### chat_ui.cpp

```cpp
bool ChatUI::Initialize() {
    screen_ = lv_scr_act();
    if (!screen_) {
        ESP_LOGE(TAG, "Failed to get active screen");
        return false;
    }
    lv_obj_set_style_bg_color(screen_, COLOR_BG, LV_PART_MAIN);

    CreateTitleBar();
    CreateStatusBar();
    CreateMessageList();
    CreateInputArea();

    // 验证关键对象
    if (!msg_list_ || !text_area_ || !send_btn_ || !status_label_) {
        ESP_LOGE(TAG, "Critical UI object creation failed");
        return false;
    }

    ESP_LOGI(TAG, "ChatUI initialized");
    return true;
}
```

### main.cpp 同步修改

```cpp
// 替换:
ui->Initialize();

// 为:
if (!ui->Initialize()) {
    ESP_LOGE(TAG, "ChatUI init failed, restarting...");
    esp_restart();
}
```

## F21 🟡 lv_timer_del 应在 lv_obj_del 之前

`ShowSplashScreen()` 约第 84-90 行:

```cpp
// ❌ 当前: 先删 splash (连带删子对象), 再删 timer
lv_timer_t* timer = lv_timer_create(
    [](lv_timer_t* t) {
        lv_obj_t* obj = static_cast<lv_obj_t*>(t->user_data);
        lv_timer_del(t);       // ← 先删 timer
        lv_obj_del(obj);       // ← 再删对象
    },
    2000, splash);
```

> 实际上当前代码已经是 lv_obj_del 在前的顺序。检查确认后，如果顺序已是 timer_del → obj_del 则无需修改。
> 如果顺序是 obj_del → timer_del，则交换。

确认: 当前代码是 `lv_obj_del(obj); lv_timer_del(t);` — 应该先 `lv_timer_del(t)` 再 `lv_obj_del(obj)`，因为 timer callback 参数 `t` 在 `lv_timer_del` 后失效。

修复:
```cpp
lv_timer_del(t);       // ← 先
lv_obj_del(obj);       // ← 后 (t->user_data 已不用)
```
